Setup NanoClass2 in amplicomics share

Published

January 16, 2024

This is a guide on how to run the NanoClass2 classifer on crunchomics. NanoClass2 is a snakemake workflow that will:

  • Remove adaptors with porechop
  • Perform quality fitering with chopper
  • Generate read quality plots post quality cleaning with pistis
  • Subsample reads to classify several classifiers (can be turned of)
  • Classify reads with any one of these tools: “blastn”,“centrifuge”,“dcmegablast”,“idtaxa”,“megablast”,“minimap”,“mothur”,“qiime”,“rdp”,“spingo”, “kraken”

All used software and its version numbers is cited here. For more information, please also have a look at the manual.

Installation

This code below was already run, and is just kept for reference:

```{bash, code-fold=“true”} #install snakemake (not run, already installed) as a conda/mamba environment mamba create -c conda-forge -c bioconda –prefix /zfs/omics/projects/amplicomics/miniconda3/envs/snakemake_nanoclass2 python=3.9.7 snakemake=6.8.0 tabulate=0.8

#go to folder in which to install software cd /zfs/omics/projects/amplicomics/bin

#install NanoClass2 git clone https://github.com/ndombrowski/NanoClass2.git





## Setup working directory

To be able to use the amplicomics share in which the sotware is installed please contact [Nina Dombrowski]( n.dombrowski@uva.nl). To be able to use software installed on the amplicomics share, you first need to set up your environment:



```{bash}
conda config --add envs_dirs /zfs/omics/projects/amplicomics/miniconda3/envs/

This will add pre-installed conda environments you your conda env list. After you run conda env list you should see several tools from amplicomics, including qiime and snakemake.

Next, you can setup your working environment:

#define the directory in which you want to run your analyses
wdir="/home/ndombro/personal/testing/amplicomics_test"

#go into the working directory
cd $wdir

#start the snakemake conda environment 
conda activate snakemake_nanoclass2

Setup a testrun

The test run will be performed with some example data provided with NanoClass2. The test run will subsample the reads and use the subsetted reads to test all tools.

#cp some test data over (comes with NanoClass2) 
#however, you can have the exact same structure, where you have your files to analyse in a data folder
mkdir data 
cp /zfs/omics/projects/amplicomics/bin/NanoClass2/example_files/*fastq.gz data/

#copy the mapping file  (comes with NanoClass2) 
cp /zfs/omics/projects/amplicomics/bin/NanoClass2/example_files/mapping.csv .

#ensure that the path to the fastq data is correct in mapping.csv
#do this with excel, nano or sed to search and replace the path
#after running this, you can do a head command to ensure that things work correctly
search='your_path/example_files/'
replace='/home/ndombro/personal/testing/amplicomics_test/data/'

sed -i -e  "s|$search|$replace|" mapping.csv

#get config yaml (comes with NanoClass2) 
cp /home/ndombro/personal/software/NanoClass2/config.yaml .

In the config yaml:

  • exchange "example_files/mapping.csv" with "mapping.csv"
  • keep rest as is, i.e. we want to subsample for the test run (this also ensures that all databases are downloaded)

By default the config.yaml is setup to run NanoClass2 as follows:

  • Subsample the fastq files to only work with 100 reads per sample. To turn this of, set skip: true
  • Run NanoClass2 on all of the following methods: [“blastn”,“centrifuge”,“dcmegablast”,“idtaxa”,“megablast”,“minimap”,“mothur”,“qiime”,“rdp”,“spingo”, “kraken”]
  • Quality filter the samples by:
    • Discard reads shorter than 1400 bp (minlen)
    • Discard reads longer than 1600bp (maxlen)
    • Discard reads with a phred score less than 10 (quality)
    • Do not trim nucleotides from the start (headcrop)
    • Do not trim nucleotides from the end (tailcrop)

Dry-run

Running snakemake with -np will not run NanoClass itself but just confirm if everything works alright. This is recommended to ensure that everything works before actually running everything.

#perform a dry-run
snakemake --cores 1 \
    -s /zfs/omics/projects/amplicomics/bin/NanoClass2/Snakefile \
    --configfile config.yaml \
    --use-conda --conda-prefix /zfs/omics/projects/amplicomics/bin/NanoClass2/.snakemake/conda \
    --nolock --rerun-incomplete -np

If the above command works and you see This was a dry-run (flag -n). The order of jobs does not reflect the order of execution. all is good and you can submit things to a compute node.

Submit sbatch job

The NanoClass2 installation comes with an example batch script that you can move over to where you do your analyses:

#cp template batch script (comes with NanoClass2) 
cp /zfs/omics/projects/amplicomics/bin/NanoClass2/jobscript.sh .

The jobscript is setup to work with NanoClass2 in the amplicomics environment and you should not need to change anything if your config.yaml is correct.

The script does the following:

  • Request 58 crunchomics cpus and 460G of memory. One can set this lower, but for testing purposes you might keep these values. The can be changed with #SBATCH --cpus-per-task=58 and #SBATCH --mem=460G
  • Ensure that conda is loaded properly by running source ~/.bashrc. The bashrc is a script executed once a user logs in and holds special configurations, such as telling the shell where conda is installed. If you run into issues that are related to conda/mamba not being found open the this file with nano ~/.bashrc and check that you have a section with text similar to # >>> conda initialize >>>. If this is not present you might need to run conda init bash to add it.
  • Activate the snakemake conda environment that is found in the amplicomics share
  • Run NanoClass2. When using the script as is, the script assumes that NanoClass2 is installed at this path /zfs/omics/projects/amplicomics/bin/NanoClass2/ and that the config.yaml is located in the folder from which you analyse the data. If that is not the case, change the paths accordingly

Next, we can submit the job:

#Submitted batch job 427090
sbatch jobscript.sh

#for checking how far things are along:
#check the end of the log file i.e. 
tail *.log 

#check if sbatch script is still running 
squeue

Generate report

snakemake --report report.html \
  --configfile config.yaml \
  -s /zfs/omics/projects/amplicomics/bin/NanoClass2/Snakefile

To view the report, its best to copy the html to your computer and open it via any web browser.

The report holds the key information, however, there are other useful outputs, all of which can be found in the results folder. The NanoClass2 website gives some more information about the generated outputs.
